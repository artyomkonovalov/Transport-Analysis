## Цель проекта

Проанализировать городскую транспортную сеть Санкт-Петербурга на примере трамвайных маршрутов с использованием графовых структур данных и алгоритмов анализа графов.

В рамках проекта транспортная сеть рассматривается как граф, где:
- вершины – остановки
- рёбра – связи между остановками в рамках маршрутов
## Задачи проекта

В рамках кейса необходимо:
1. собрать и обработать реальные открытые данные о городском транспорте
2. спроектировать собственные структуры данных для хранения информации об остановках и маршрутах
3. построить граф транспортной сети на основе маршрутов
4. выполнить анализ транспортной сети: 
	- выделить топ-маршруты
	- ранжировать маршруты по длине (включению остановок)
	- исследовать граф сети на связность
5. подготовить визуализации с использованием Yandex DataLens
6. оформить результаты в виде работающего проекта и отчёта

## Анализируемые данные

### Остановки

Для каждой остановки анализируются следующие поля:
1. **ID остановки** *(текcтовое поле)*
2. **Название остановки** *(текcтовое поле)*
3. **Координаты**:
    - широта (latitude) *(дробное число)*
    - долгота (longitude) *(дробное число)*
4. **\*Район города** *(текстовое поле Enum)*

После обработки данных для остановок дополнительно вычисляется:
+ **Количество маршрутов, проходящих через остановку** *(целое число)*
### Маршруты

Для каждого маршрута анализируются следующие поля:
1. **ID маршрута** *(текcтовое поле)*
2. **Номер маршрута** *(текcтовое поле)*
3. **Тип транспорта** *(текстовое поле Enum)*

После обработки данных для маршрутов дополнительно вычисляется:
+ **Количество остановок в маршруте** *(целое число)*
+ **Длина маршрута в км** *(дробное число)*

### Описание маршрутов

Граф транспортной сети строится на основе описания маршрутов, которое включает:
1. ID маршрута *(текcтовое поле)*
2. ID остановки *(текcтовое поле)*
3. порядок следования остановок внутри маршрута *(порядковый номер)*

На основе этих данных формируются рёбра графа между соседними остановками.

## Анализ транспортной сети

После построения графа выполняется анализ с целью получения качественных ответов на следующий вопросы:

1. **Топ остановок по числу маршрутов**  
    Какие 5–10 остановок обслуживают наибольшее количество маршрутов?

2. **Длины маршрутов**  
    Какой маршрут является самым длинным и самым коротким по числу остановок?

3. **Связность сети**  
    Сколько компонент связности содержит граф транспортной сети?

## Визуализация

Для визуального анализа данных используются инструменты **Yandex DataLens**.

Планируемые параметры визуализации:
- карта города с нанесёнными остановками
- каждая остановка отображается узлом
- цвет или размер узла отражает количество маршрутов, проходящих через остановку

## Источники данных

Для проекта используются открытые GTFS-данные *(General Transit Feed Specification)*, опубликованные на [государственном портале открытых данных Санкт-Петербурга](https://data.gov.spb.ru/).

**Используемые ресурсы:**

**[1] [Маршруты движения городского транспорта (трамваи)](https://data.gov.spb.ru/irsi/7830001067-marshruty-dvizheniya-gorodskogo-transporta/)**

**[2] [Перечень остановочных пунктов с координатами](https://data.gov.spb.ru/irsi/7830001067-perechen-ostanovochnyh-punktov-s-ukazaniem-vida-transporta-i-s-koordinatami-ih-mestopolozheniya/)**

**[3] [Распакованный архив с сырыми данными, выгруженный с data.gov.spb.ru](https://github.com/artyomkonovalov/Transport-Analysis/tree/main/data/raw_data_gov_spb/)**
